# ROADMAP.md

## 一、当前阶段判断

Financial Research Assistant 已经从“功能堆叠阶段”进入“研究系统化阶段”。

前期主要在做：
- 页面搭建
- 基本链路打通
- JSON 结构沉淀
- 模型接入层抽象

当前进入的新阶段是：
- 运行态工程化
- 时间序列一致性治理
- 公司画像体系增强
- 预测/回测闭环强化
- 正式版工作台能力提升

这意味着后续最值钱的工作，已经不是继续加页面，而是提高系统运行质量与研究一致性。

## 三、当前最重要的未完成事项

### P0：运行态工程化
这是当前最高优先级之一。

#### 目标
让正式版工作台和核心链路具备：
- 后台任务执行
- 任务状态可见
- 进度条
- 日志输出
- 协作式取消
- 未来可扩展为真正强中断

#### 当前问题
- 取消仍不是硬中断
- 状态文件写入在 Windows 上仍可能有冲突风险
- 长任务仍可能让用户感知“卡住”

#### 后续工作
- 统一 task registry
- 引入更稳定的任务状态机
- 支持断点恢复
- 支持失败任务重跑
- 支持单步任务诊断

---

### P0：parse 深度优化
parse 是当前整个系统最重要也最容易成为瓶颈的部分。

#### 当前问题
- 某些 PDF 解析仍然很慢
- 缺少单文件耗时追踪
- 缺少单页耗时追踪
- 缺少“哪个引擎慢”的诊断信息
- 复杂页多模态策略仍需优化

#### 后续工作
- 单文件耗时 profiler
- 单页耗时 profiler
- 多模态页统计
- 文本补救页统计
- 大文件分批解析
- parse 任务可恢复
- 慢页跳过与降级策略

---

### P0：主财报 / 辅助材料分离治理
这是系统正确性的底线。

#### 当前问题
- 某些业绩公告/演示材料可能被误识别为主财报
- 历史旧文件可能已经写入错误 document_type
- 辅助材料与主时序混用的风险依然存在

#### 后续工作
- 进一步强化 document_type 判定规则
- 增加旧数据修复脚本
- 增加 repository 中主/辅材料显式展示
- 增加 actuals 写入前强校验

---

### P1：公司画像与行业 profile 深化
公司画像是整个研究系统的重要上层逻辑。

#### 当前问题
- 全行业覆盖仍不完整
- 某些公司标签容易误判
- 需要更强的联网核验支持
- 自动画像与手动画像之间的冲突处理仍可优化

#### 后续工作
- 扩展行业 profile 至更完整行业谱系
- 增强 BM / VC / LC / G 标签规则
- 引入 LC 的更细 sub_type 体系
- 公司画像联网校验按钮
- 人工确认工作流
- 历史画像变更日志

---

### P1：update / forecast / backtest 的状态总线统一
当前系统虽然已有大量 registry，但还没有一套真正统一的状态总线。

#### 建议建立的 registry
- material_ledger
- timeline_registry
- analysis_snapshot_registry
- forecast_backtest_registry
- management_feedback_registry
- workflow_stage_registry

#### 目标
把“很多页面 + 很多 JSON”进一步收敛成“系统级状态契约”。

---

### P1：模型输出 JSON 稳定性增强
#### 当前问题
- 模型返回脏 JSON
- repair 流程还不够强
- schema 约束还不够刚性

#### 后续工作
- 更严格的 JSON schema
- 更强的 repair agent
- 本地 fallback parser
- 输出片段回显优化
- 对 analyze / update / summary 等输出层统一做 JSON 验证

---

### P2：正式版工作台完善
#### 当前目标
正式版工作台应逐步承担：
- 一键入库
- 一键分析
- 一键更新研究报告
- 一键预测
- 一键回测

#### 当前问题
- 正式版流程仍不完整
- 某些高级功能仍需回到测试版
- 任务运行态信息还不够全面

#### 后续工作
- 增加正式版 update 工作流
- 增加正式版 forecast / forecast_check 工作流
- 增加任务中心页
- 增加失败重试与状态回看

---

### P2：QNAP 迁移与部署治理
#### 当前目标
支持从本地开发环境迁移到 QNAP 等本地云环境。

#### 当前问题
- 路径虽然开始抽象，但迁移流程仍未正式固化
- 数据目录、日志目录、任务目录还需统一

#### 后续工作
- QNAP 部署文档
- 环境变量模板
- 数据目录迁移脚本
- 运行状态目录规范

---

### 第二阶段

- 提升正式版体验
- 让日常研究工作真正跑在正式版

- 完善文档
- 建立测试集和回归测试

## 五、哪些工作现在不要急着做

以下事项不是不重要，而是当前不应优先：

1. 过度美化前端
2. 过度追求某个模型的“回答更像人”
3. 为了展示效果牺牲主/辅材料边界
4. 在 registry 尚未统一前频繁大改 JSON 结构
5. 过早做复杂权限系统

当前最重要的是：
- 跑稳
- 可追踪
- 可解释
- 可回测
- 不污染主时序
